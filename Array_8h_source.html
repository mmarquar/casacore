58 // <summary> A templated N-D Array class with zero origin </summary>
59 
60 // Array<T> is a templated, N-dimensional, Array class. The origin is zero,
61 // but by default indices are zero-based. This Array class is the
62 // base class for specialized Vector<T>, Matrix<T>, and Cube<T> classes.
63 //
64 // Indexing into the array, and positions in general, are given with IPosition
65 // (essentially a vector of integers) objects. That is, an N-dimensional
66 // array requires a length-N IPosition to define a position within the array.
67 // Unlike C, indexing is done with (), not []. Also, the storage order
68 // is the same as in FORTRAN, i.e. memory varies most rapidly with the first
69 // index.
70 // <srcblock>
71 // // axisLengths = [1,2,3,4,5]
72 // IPosition axisLengths(5, 1, 2, 3, 4, 5);
73 // Array<Int> ai(axisLengths); // ai is a 5 dimensional array of
74 // // integers; indices are 0-based
75 // // => ai.nelements() == 120
76 // Array<Int> ai2(axisLengths); // The first element is at index 0
77 // IPosition zero(5); zero = 0; // [0,0,0,0,0]
78 // //...
79 // </srcblock>
80 // Indexing into an N-dimensional array is relatively expensive. Normally
81 // you will index into a Vector, Matrix, or Cube. These may be obtained from
82 // an N-dimensional array by creating a reference, or by using an
83 // ArrayIterator. The "shape" of the array is an IPosition which gives the
84 // length of each axis.
85 //
86 // An Array may be standalone, or it may refer to another array, or to
87 // part of another array (by refer we mean that if you change a pixel in
88 // the current array, a pixel in the referred to array also changes, i.e.
89 // they share underlying storage).
90 // <note role=warning>
91 // One way one array can reference another is through the copy
92 // constructor. While this might be what you want, you should
93 // probably use the reference() member function to make it explicit.
94 // The copy constructor is used when arguments are passed by value;
95 // normally functions should not pass Arrays by value, rather they
96 // should pass a reference or a const reference. On the positive
97 // side, returning an array from a function is efficient since no
98 // copying need be done.
99 // </note>
100 //
101 // Aside from the explicit reference() member function, a user will
102 // most commonly encounter an array which references another array
103 // when he takes an array slice (or section). A slice is a sub-region of
104 // an array (which might also have a stride: every nth row, every mth column,
105 // ...).
106 // <srcblock>
107 // IPosition lengths(3,10,20,30);
108 // Array<Int> ai(lengths); // A 10x20x30 cube
109 // Cube<Int> ci;
110 // //...
111 // ci.reference(ai1); // ci and ai now reference the same
112 // // storage
113 // ci(0,0,0) = 123; // Can use Cube indexing
114 // ci.xyPlane(2) = 0; // and other member functions
115 // IPosition zero(3,0,0,0);
116 // assert(ai(zero) == 123); // True because ai, ci are references
117 // //...
118 // Array<Int> subArray;
119 // IPosition blc(3,0,0,0), trc(3,5,5,5);
120 // subArray.reference(ai(blc, trc));
121 // subArray = 10; // All of subArray, which is the
122 // // subcube from 0,0,0 to 5,5,5 in
123 // // ai, has the value 10.
124 // </srcblock>
125 // While the last example has an array slice referenced explicitly by another
126 // array variable, normally the user will often only use the slice as
127 // a temporary in an expresion, for example:
128 // <srcblock>
129 // Array<Complex> array;
130 // IPosition blc, trc, offset;
131 // //...
132 // // Copy from one region of the array into another
133 // array(blc, trc) = array(blc+offset, trc+offset);
134 // </srcblock>
135 //
136 // The Array classes are intended to operate on relatively large
137 // amounts of data. While they haven't been extensively tuned yet,
138 // they are relatively efficient in terms of speed. Presently they
139 // are not space efficient -- the overhead is about 15 words. While
140 // this will be improved (probably to about 1/2 that), these array
141 // classes are not appropriate for very large numbers of very small
142 // arrays. The Block<T> class may be what you want in this circumstance.
143 //
144 // Element by element mathematical and logical operations are available
145 // for arrays (defined in aips/ArrayMath.h and aips/ArrayLogical.h).
146 // Because arithmetic and logical functions are split out, it is possible
147 // to create an Array<T> (and hence Vector<T> etc) for any type T that has
148 // a default constructor, assignment operator, and copy constructor. In
149 // particular, Array<String> works.
150 //
151 // If compiled with the preprocessor symbol AIPS_DEBUG symbol, array
152 // consistency ("invariants") will be checked in most member
153 // functions, and indexing will be range-checked. This should not be
154 // defined for production runs.
155 //
156 // <note role=tip>
157 // Most of the data members and functions which are "protected" should
158 // likely become "private".
159 // </note>
160 //
161 // <todo asof="1999/12/30">
162 // <li> Integrate into the Lattice hierarchy
163 // <li> Factor out the common functions (shape etc) into a type-independent
164 // base class.
165 // </todo>

702  // Define the STL-style iterator functions (only forward iterator).
703  // It makes it possible to iterate through all data elements of an array
704  // and to use it common STL functions.
705  // The end() function is relatively expensive, so it should not be
706  // used inside a for statement. It is much better to call it beforehand
707  // as shown in the example below. Furthermore it is very important to
708  // use <src>++iter</src>, because <src>iter++</src> is 4 times slower.
709  // <srcblock>
710  // Array<Int> arr(shape);
711  // Array<Int>::iterator iterend(arr.end());
712  // for (Array<Int>::iterator iter=arr.begin(); iter!=iterend; ++iter) {
713  // *iter += 1;
714  // }
715  // </srcblock>
716  // The Array class supports random access, so in principle a random
717  // iterator could be implemented, but its performance would not be great,
718  // especially for non-contiguous arrays.

720  // If the array is contiguous, it is possible to use the
