Installing AIPS++

Warning: At this stage of development of AIPS++, it is available only via anonymous ftp. This document describes how to install the development system, which allows coding in AIPS++. Since the system is currently very volatile, we discourage you from trying to use it unless you have a good reason, and have talked to Project staff beforehand. We offer no support for use of the development system. If coding is not important, then we very strongly prefer that you wait to use one of the official beta releases, the next of which will be available in June 1999. We do provide support for the beta releases, provided that you are a member of our beta-tester program. In either event, please first contact the AIPS++ Project Manager Tim Cornwell before downloading.

Installation of the AIPS++ development system is quite simple. You download and install a bootstrap that contains scripts to set up the necessary environment and then inhale the complete system. The bootstrap phase takes a few minutes, and the inhale typically takes many hours. The inhale ftp's a total of about 50 MB and then builds the system. The space required for a full installation is about 500MB. The one likely complication is that AIPS++ relies upon some other packages and compilers to be present. Copies of the required packages are kept in the ftp area in pub/import and are fetched by the configure script (see below) if necessary. More information can be found in the AIPS++ System Manual.

In the following, commands to feed to your shell are in this font and also are prefixed by $'s:

1) Go to where you want the AIPS++ installation's root area to be located (e.g. /aips++/), type

$ mkdir code ; cd code
2) FTP to aips2.nrao.edu:/pub/RELEASED/latest and get the following files

  • configure
  • install-11.000.tar.gz
  • VERSION

3) After you receive the files,

$ chmod +x configure

$ ./configure --source

Answer the questions configure gives you. Edit items like the aipsrc and makedefs files as you see fit. Remember to put something sensible in the printer* entries in the aipsrc file .

4) If you're running the sh/ksh/bash shell, type

$ . aipsinit.sh

or, if you're running a csh/tcsh flavored shell, type

$ source aipsinit.csh
You now have the bootstrap part of the system loaded.

5) You can now get the latest version of the entire system automatically via a cumulative inhale by typing

$ inhale -c

The last step takes some time (a few to perhaps 10-15 hours). When finished, you should have a working, up-to-date, AIPS++ installation.

More information on the AIPS++ code management and distribution system can be found by accessing AIPS++ Source Code and in the System Reference Manual.
