59 // <summary>
60 // Interconvert between Casacore Images and FITS files.
61 // </summary>
62 
63 // <use visibility=export>
64 
65 // <reviewed reviewer="" date="yyyy/mm/dd" tests="" demos="">
66 // </reviewed>
67 
68 // <prerequisite>
69 // <li> <linkto class="PagedImage">PagedImage</linkto>
70 // <li> <linkto class="PrimaryArray">PrimaryArray</linkto> (and FITS concepts in
71 // general).
72 // </prerequisite>
73 //
74 // <synopsis>
75 // This class is a helper class that is used to interconvert between Casacore
76 // images and FITS files. This adds no functionality over the general abilities
77 // available in the underlying FITS classes, however it is a useful higher-level
78 // packaging.
79 //
80 // There are two fundamental member functions in this class.
81 // <src>FITSToImage</src> which turns a FITS file into a Casacore image, and
82 // <src>ImageToFITS</src> which does the opposite.
83 //
84 // We can read images from any HDU inside the FITS file (although this isn't
85 // well tested). Images with a quality axis (i.e. contain data and error values)
86 // are stored in the primary HDU (data) and an extension HDU (error). Other
87 // images are always written to the primary HDU.
88 //
89 // Pixels in the FITS file which are blanked are masked out (the mask
90 // is set to False) in the output image. On conversion to FITS,
91 // masked values are blanked. The mask which is read is the current
92 // default mask.
93 // </synopsis>
94 //
95 // <example>
96 // A FITS to image conversion may be accomplished as follows:
97 // <srcBlock>
98 // PagedImage<Float> *image = 0;
99 // String fitsName = "exists.fits";
100 // String imageName = "new.image";
101 // String error;
102 // Bool ok = ImageFITSConverter::FITSToImage(image, error, imageName, fitsName);
103 // if (!image) ... error ...
104 // </srcBlock>
105 // A couple of things to note:
106 // <ul>
107 // <li> If <src>ok</src> is False, the conversion failed and <src>error</src>
108 // will be set.
109 // <li> The pointer "image" is set if the conversion succeeds. If it is
110 // zero the conversion failed and <src>error</src> will contain an
111 // error message.
112 // <li> The caller is responsible for deleting the pointer <src>image</src>
113 // when the conversion is successful.
114 // </ul>
115 // Similarly, an image to FITS conversion may be accomplished as follows:
116 // <srcBlock>
117 // String imageName = argv[1];
118 // PagedImage<Float> image = ...; // An existing image from somewhere
119 // String fitsName = "new.fits";
120 // String error;
121 // Bool ok = ImageFITSConverter::ImageToFITS(error, image, fitsName);
122 // </srcBlock>
123 // A couple of similar remarks can be made about this example:
124 // <ul>
125 // <li> If <src>ok</src> is False, the conversion failed and <src>error</src>
126 // will be set.
127 // </ul>
128 // </example>
129 //
130 // <motivation>
131 // FITS files are the fundamental transport format for images in Astronomy.
132 // </motivation>
133 //
134 // <todo asof="1999/02/15">
135 // <li> It might be useful to have functions that convert between FITS
136 // and general lattices.
137 // <li> Add support for PagedImage<Complex>
138 // <li> Convert multiple images at once?
139 // <li> Allow writing FITS files to an image extension in an existing
140 // FITS file.
141 // </todo>

192  // Convert a Casacore image to a FITS file.
193  // <ul>
194  // <li> <src>return</src> True if the conversion succeeds, False
195  // otherwise.
196  // <li> <src>error</src> will be set if the conversion fails.
197  // <li> <src>image</src> The image to convert.
198  // <li> <src>fitsName</src> If the name is "-" (the minus character),
199  // then write to stdout Always writes to the primary array.
200  // <li> <src>memoryInMB</src>. Setting this to zero will result in
201  // row-by-row copying, otherwise it will attempt to with as large
202  // a chunk-size as possible, while fitting in the desired memory.
203  // <li> <src>preferVelocity</src>Write a velocity primary spectral axis
204  // if possible.
205  // <li> <src>opticalVelocity</src>If writing a velocity, use the optical
206  // definition (otherwise use radio).
207  // <li> <src>BITPIX, minPix, maxPix</src>
208  // BITPIX can presently be set to -32 or 16 only. When BITPIX is
209  // 16 it will write BSCALE and BZERO into the FITS file. If minPix
210  // is greater than maxPix the minimum and maximum pixel values
211  // will be determined from the array, otherwise the supplied
212  // values will be used and pixels outside that range will be
213  // truncated to the minimum and maximum pixel values (note that
214  // this truncation does not occur for BITPIX=-32).
215  // <li> <src>allowOverwrite</src> If True, allow fitsName to be
216  // overwritten if it already exists.
217  // <li> <src>degenerateLast</src> If True, axes of length 1 will be written
218  // last to the header.
219  // <li> <src>preferWavelength</src> If True, write a wavelength primary axis.
220  // <li> <src>airWavelength</src> If True and <src>preferWavelength</src> is True write
221  // an air wavelength primary axis.
222  // <li> <src>origin</src> gives the origin, i.e., the name of the package.
223  // If empty, it defaults to "casacore-"getVersion().
224  // </ul>
