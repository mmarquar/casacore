54  // <group>
55  // In the following group of methods, if the size of the composite dataset
56  // is smaller than
57  // <src>binningThreshholdSizeBytes</src>, the composite dataset
58  // will be (perhaps partially) sorted and persisted in memory during the
59  // call. In that case, and if <src>persistSortedArray</src> is True, this
60  // sorted array will remain in memory after the call and will be used on
61  // subsequent calls of this method when <src>binningThreshholdSizeBytes</src>
62  // is greater than the size of the composite dataset. If
63  // <src>persistSortedArray</src> is False, the sorted array will not be
64  // stored after this call completes and so any subsequent calls for which the
65  // dataset size is less than <src>binningThreshholdSizeBytes</src>, the
66  // dataset will be sorted from scratch. Values which are not included due to
67  // non-unity strides, are not included in any specified ranges, are masked,
68  // or have associated weights of zero are not considered as dataset members
69  // for quantile computations.
70  // If one has a priori information regarding
71  // the number of points (npts) and/or the minimum and maximum values of the data
72  // set, these can be supplied to improve performance. Note however, that if these
73  // values are not correct, the resulting median
74  // and/or quantile values will also not be correct (although see the following notes regarding
75  // max/min). Note that if this object has already had getStatistics()
76  // called, and the min and max were calculated, there is no need to pass these values in
77  // as they have been stored internally and used (although passing them in shouldn't hurt
78  // anything). If provided, npts, the number of points falling in the specified ranges which are
79  // not masked and have weights > 0, should be exactly correct. <src>min</src> can be less than
80  // the true minimum, and <src>max</src> can be greater than the True maximum, but for best
81  // performance, these should be as close to the actual min and max as possible.
82  // In order for quantile computations to occur over multiple datasets, all datasets
83  // must be available. This means that if setCalculateAsAdded()
84  // was previously called by passing in a value of True, these methods will throw
85  // an exception as the previous call indicates that there is no guarantee that
86  // all datasets will be available. If one uses a data provider (by having called
87  // setDataProvider()), then this should not be an issue.

90  // For a dataset with an odd number of good points, the median is just the value
91  // at index int(N/2) in the equivalent sorted dataset, where N is the number of points.

93  // indices int(N/2)-1 and int(N/2) in the sorted dataset.
