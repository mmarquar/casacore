43 // Class to calculate statistics in a "classical" sense, ie using accumulators with no
44 // special filtering beyond optional range filtering etc.
45 //
46 // setCalculateAsAdded() allows one to specify if statistics should be calculated and updated
47 // on upon each call to set/addData(). If False, statistics will be calculated only when
48 // getStatistic(), getStatistics(), or similar methods are called. Setting this value to True
49 // allows the caller to not have to keep all the data accessible at once. Note however, that all
50 // data must be simultaneously accessible if quantile (eg median) calculations are desired.
51 
52 // I attempted to write this class using the Composite design pattern, with eg the
53 // _unweightedStats() and _weightedStats() methods in their own class, but for reasons I
54 // don't understand, that impacted performance significantly. So I'm using the current
55 // architecture, which I know is a bit a maintenance nightmare.

79  // In the following group of methods, if the size of the composite dataset
80  // is smaller than
81  // <src>binningThreshholdSizeBytes</src>, the composite dataset
82  // will be (perhaps partially) sorted and persisted in memory during the
83  // call. In that case, and if <src>persistSortedArray</src> is True, this
84  // sorted array will remain in memory after the call and will be used on
85  // subsequent calls of this method when <src>binningThreshholdSizeBytes</src>
86  // is greater than the size of the composite dataset. If
87  // <src>persistSortedArray</src> is False, the sorted array will not be
88  // stored after this call completes and so any subsequent calls for which the
89  // dataset size is less than <src>binningThreshholdSizeBytes</src>, the
90  // dataset will be sorted from scratch. Values which are not included due to
91  // non-unity strides, are not included in any specified ranges, are masked,
92  // or have associated weights of zero are not considered as dataset members
93  // for quantile computations.
94  // If one has a priori information regarding the number of points (npts) and/or
95  // the minimum and maximum values of the data set, these can be supplied to
96  // improve performance. Note however, that if these values are not correct, the
97  // resulting median
98  // and/or quantile values will also not be correct (although see the following notes regarding
99  // max/min). Note that if this object has already had getStatistics()
100  // called, and the min and max were calculated, there is no need to pass these values in
101  // as they have been stored internally and used (although passing them in shouldn't hurt
102  // anything). If provided, npts, the number of points falling in the specified ranges which are
103  // not masked and have weights > 0, should be exactly correct. <src>min</src> can be less than
104  // the true minimum, and <src>max</src> can be greater than the True maximum, but for best
105  // performance, these should be as close to the actual min and max as possible.
106  // In order for quantile computations to occur over multiple datasets, all datasets
107  // must be available. This means that if setCalculateAsAdded()
108  // was previously called by passing in a value of True, these methods will throw
109  // an exception as the previous call indicates that there is no guarantee that
110  // all datasets will be available. If one uses a data provider (by having called
111  // setDataProvider()), then this should not be an issue.

113  // get the median of the distribution.
114  // For a dataset with an odd number of good points, the median is just the value
115  // at index int(N/2) in the equivalent sorted dataset, where N is the number of points.
116  // For a dataset with an even number of points, the median is the mean of the values at
117  // indices int(N/2)-1 and int(N/2) in the sorted dataset.
118  virtual AccumType getMedian(
119  CountedPtr<uInt64> knownNpts=NULL, CountedPtr<AccumType> knownMin=NULL,
120  CountedPtr<AccumType> knownMax=NULL, uInt binningThreshholdSizeBytes=4096*4096,
121  Bool persistSortedArray=False
122  );

124  // If one needs to compute both the median and quantile values, it is better to call
125  // getMedianAndQuantiles() rather than getMedian() and getQuantiles() separately, as the
126  // first will scan large data sets fewer times than calling the separate methods.
127  // The return value is the median; the quantiles are returned in the <src>quantiles</src> map.
