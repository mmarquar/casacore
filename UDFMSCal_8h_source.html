54 // <synopsis>
55 // UDFMSCal defines UDFs (user defined functions) that can be used in TaQL
56 // to get derived MeasurementSet values hourangle, parallactic angle,
57 // azimuth/elevation, and local sidereal time.
58 // In this way such derived values appear to be ordinary TaQL functions.
59 //
60 // The following functions can be defined:
61 // <ul>
62 // <li> HA is the hourangle of the array center (observatory position).
63 // <li> HA1 is the hourangle of ANTENNA1.
64 // <li> HA2 is the hourangle of ANTENNA2.
65 // <li> HADEC is the hourangle/DEC of the array center (observatory position).
66 // <li> HADEC1 is the hourangle/DEC of ANTENNA1.
67 // <li> HADEC2 is the hourangle/DEC of ANTENNA2.
68 // <li> LAST is the local sidereal time of the array center.
69 // <li> LAST1 is the local sidereal time of ANTENNA1.
70 // <li> LAST2 is the local sidereal time of ANTENNA2.
71 // <li> PA1 is the parallactic angle of ANTENNA1.
72 // <li> PA2 is the parallactic angle of ANTENNA2.
73 // <li> AZEL1 is the azimuth/elevation of ANTENNA1.
74 // <li> AZEL2 is the azimuth/elevation of ANTENNA2.
75 // <li> UVW_J2000 is the UVW coordinates in J2000 (in meters)
76 // <li> STOKES makes it possible to convert Stokes of data, flag, or weight.
77 // <li> BASELINE is baseline selection using CASA syntax.
78 // <li> TIME is baseline selection using CASA syntax.
79 // <li> UVDIST is UV-distance selection using CASA syntax.
80 // <li> SPW is spectral window selection using CASA syntax.
81 // <li> FIELD is field selection using CASA syntax.
82 // <li> ARRAY is array selection using CASA syntax.
83 // <li> SCAN is scan selection using CASA syntax.
84 // <li> STATE is state selection using CASA syntax.
85 // <li> OBS is observation selection using CASA syntax.
86 // </ul>
87 // The first functions have data type double and unit radian (except UVW).
88 // The HADEC, AZEL, and UVW functions return arrays while the others return
89 // scalars.
90 // <br>The STOKES function can have data type Complex, Double or Bool.
91 // <br>The latter functions are selection functions and return a Bool scalar.
92 //
93 // This class is meant for a MeasurementSet, but can be used for any table
94 // containing an ANTENNA and FIELD subtable and the relevant columns in the
95 // main table (ANTENNA1 and/or ANTENNA2, FIELD_ID, and TIME).
96 // <br>In principle the array center is the Observatory position, which is
97 // taken from the Measures Observatory table using the telescope name found
98 // in the OBSERVATION subtable. However, if the subtable is not defined or
99 // empty or if the telescope name is unknown, the position of the first antenna
100 // is used as the array position.
101 //
102 // The engine can also be used for a CASA Calibration Table. It understands
103 // how it references the MeasurementSets. Because calibration tables contain
104 // no ANTENNA2 columns, functions XX2 are the same as XX1.
105 // </synopsis>
106 
107 // <motivation>
108 // It makes it possible to do queries on these values without having
109 // to add columns for them.
110 // </motivation>
