56 // <prerequisite>
57 // <li> <linkto class="MeasurementSet">MeasurementSet</linkto> module
58 // </prerequisite>
59 //
60 // <etymology>
61 // From "MeasurementSet" and "selection".
62 // </etymology>
63 //
64 // <synopsis>
65 // The MSSelection class represents a selection on a MeasurementSet (MS).
66 // This class is used in translating MS selections represented as
67 // selection items in the user interface, and for converting between
68 // MS selection and pure TaQL selection.
69 //
70 // The purpose of this class is to provides a simple expression based
71 // selection mechanism to both the end-user and developer wishing to
72 // perform query operations over a measurement set. This is
73 // accomplished by abstracting the TaQL interface through an
74 // adapter/translation interface which converts STaQL (Simple Table
75 // Query Language) expressions into the equivalent table expression
76 // form, reducing the knowledge necessary to perform powerful query
77 // operations directly in TaQL. It is also possible to supply pure
78 // TaQL expression(s) as sub-expressions if required. For a complete
79 // list of the STaQL interface refer to the MeasurementSet Selection Syntax document at: <a
80 // href="http://casa.nrao.edu/other_doc.shtml">Data
81 // Selection</a>
82 //
83 // The sub-expressions are interpreted in the order which they were
84 // set. The order however in not important - any dependency on the
85 // order in which the expressions are evaluated is handled internally.
86 // The result of parsing the expressions is TableExprNode (TEN). All
87 // TENs from sub-expressions are finally ANDed and the resultant TEN
88 // is used to select the rows of the MS table.
89 //
90 // </synopsis>
91 //
92 // <example>
93 // <srcblock>
94 // // Create a MS and a MS selection
95 // MeasurementSet ms(msName);
96 // MSSelection select;
97 // // Setup any sub-expressions of interest directly
98 // // (or optionally send this information through a Record)
99 // select.setFieldExpr("0,1");
100 // select.setSpwExpr(">0");
101 // // Create a table expression over a MS representing the selection
102 // TableExprNode node = select.toTableExprNode(&ms);
103 // // Optionally create a table and new MS based on this node
104 // Table tablesel(ms.tableName(), Table::Update);
105 // MeasurementSet mssel(tablesel(node, node.nrow()));
106 // </srcblock>
107 // </example>
108 //
109 // <motivation>
110 // This class is used by the MS access classes.
111 // </motivation>
112 //
113 // <todo asof="Aug/14/2009">
114 // Generalize SpwExpressions and PolnExpressions to optionally include
115 // DataDescription ID specifications.
116 // </todo>

330  //
331  // The key in the ordered map returned by getCorrMap() is the
332  // pol. in the Data Description ID (DDID) sub-table. The value is
333  // a Vector of two Vectors.
334  //
335  // The returned Map<T> has a key that maps to two vectors:
336  // Key ----> Vector1 Vector2
337  //
338  // Key : Row index in the POLARIZATION sub-table
339  //
340  // Vector1 : List of poln. indices selected from the row pointed
341  // by Key. These are the in-row indices to pick-out the
342  // desired (selected) polarization products from the
343  // selected rows of the MS (or equivalently, the list of
344  // indices for the vector in the corrType column of the
345  // POLARIZATION sub-table).
346  //
347  // Vector2 : List of selected rows from the DATA_DESCRIPTION sub-table
348  //
349  // An example: following are the sub-tables used for the example
350  // explaination below:
351  //
352  // POLARIZATION Sub-table
353  // ======================
354  // Row Poln
355  // ------------
356  // 0 RR, LL
357  // 1 RR, LR, RL, LL
358  //
359  // DATA_DESCRIPTION Sub-table
360  // ==========================
361  // Row PolnID SpwID
362  // ------------------------------
363  // 0 0 0
364  // 1 1 1
365  // 2 1 2
366  // 3 1 3
367  // 4 1 4
368  // 5 1 5
369  // 6 1 6
370  // 7 1 7
371  // 8 1 8
372  //
373  //
374  // E.g., the expression poln='LL'
375  //
376  // returns the Map:
377  //
378  // corrmap = (0, [[1], [0]]) (1, [[3], [0,1,2,3,4,5,6,7,8]] )
379  //
380  // The rows from the POLARIZATION table selected are 0 and 1, These are
381  // two keys for the two entries in the map.
382  //
383  // 1. The two vectors in map 1 are: [1] and [0]. The this reads as:
384  // From the 0th. row of the POLARIZATION table, use the indices [1]. The
385  // relevant list of associated DD rows are [0]
386  //
387  // 2. The two vectors in map 2 are: [3] and [0,1,2,3,4,5,6,7,8]. This reads as:
388  // From the 1st. row of the POLARIZATION table, use the indices [3]. The
389  // relevant list of associated DD rows are [0,1,2,3,4,5,6,7,8].
390  //
391  // For a client code:
392  //
393  // o To get a list of the DDIDs selected, iterate over all entries of the
394  // map and collate the second vector from each entry.
395  //
396  // Or, use getDDIDList().
397  //
398  // o To get the list of the selected poln. *in-row indices*, collate the
399  // first vector from each entry.
400  //
401  // o To get a list of POLARIZATION IDs selected (rows of the POLARIZATION
402  // table), make a list of all the keys of this map.
403  inline OrderedMap<Int, Vector<Vector<Int> > > getCorrMap(const MeasurementSet* ms=NULL)
404  {getTEN(ms); return selectedSetupMap_p;};
