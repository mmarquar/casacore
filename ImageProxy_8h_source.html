49  // <synopsis>
50  // ImageProxy is a proxy to an image having data type Float, Double,
51  // Complex, or DComplex. Its primary purpose is to be bind the images
52  // module to Python through pyrap.images. However, it can also be used
53  // directly in C++.
54  //
55  // An ImageProxy object can be constructed for an image stored on disk in
56  // Casacore, FITS, HDF5, or Miriad format. It can also be constructed given
57  // a shape or an N-dim array with values.
58  // Furthermore it can be constructed from a LEL expression (see class ImageExpr)
59  // or a vector of images to be concatenated (see class ImageConcat).
60  //
61  // Many functions exist to operate on an ImageProxy object. For example:
62  // <ul>
63  // <li> get meta info (shape, data type, coordinates, etc.)
64  // <li> save in Casacore, HDF5, or FITS format.
65  // <li> regrid.
66  // <li> get statistics.
67  // <li> form a subimage (which is done in a virtual way).
68  // </ul>
69  // Functions regrid and statistics can only be used for Float images.
70  // They throw an exception for images with other data types.
71  // Note that using a LEL expression it is possible to (virtually) convert an
72  // image with another type to a Float image.
73  // </synopsis>
