55 // <etymology>
56 // TableCache represents a cache of open tables.
57 // </etymology>
58 
59 // <synopsis>
60 // A TableCache object keeps track of the tables which have already
61 // been opened in a program. It maps the name of a table to its
62 // PlainTable object.
63 // In principle only one TableCache object (statically defined in
64 // class PlainTable) exists in a process.
65 // The cache is used to prevent a table from being opened more than
66 // once, which is not only a waste of space, but more importantly,
67 // may give rise to synchronization problems.
68 // Synchronization between the same table in multiple processes must
69 // be done by a locking mechanism.
70 //
71 // TableCache is used by class Table and PlainTable.
72 // Before opening a table, Table will first look in the cache.
73 // Newly opened or created tables will be added to the cache.
74 // When a table is actually closed, it will be removed from the cache.
75 // </synopsis>
76 
77 // <motivation>
78 // When a RefTable is read back, it will also read back the table it
79 // references. However, that table may have been opened before and
80 // it is bad to have a table open more than once in the same program.
81 // The TableCache class catches this and will not reopen the table.
82 // </motivation>
83
